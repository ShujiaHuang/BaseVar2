# BaseVar v2.6.1 Release Notes

## Overview

v2.6.1 is a critical bug fix release that resolves Indel variant loss introduced by the binary batchfile format (v2.6.0). Approximately 2.2% of Indel variants were silently dropped during variant calling due to incorrect `ref_base` storage semantics in the `.bbf` binary format.

## Bug Fix

### Fix: Per-Read `ref_base` Storage in Binary Batchfile Format

**Problem**: The binary batchfile format (`.bbf`) stored `ref_base` as a position-level field — a single value shared by all reads at a given genomic position. However, after `_seek_position`'s Indel relocation, reads at the same position can have different `ref_base` lengths:

- SNP reads: `ref_base = "A"` (single base)
- Deletion reads: `ref_base = "AG"` (anchor base + deleted base)
- Insertion reads: `ref_base = "A"` (anchor base only)

When the position-level `ref_base` was taken from a non-Indel read (e.g., `"A"`), all Indel reads inherited this incorrect shorter value. This caused `collect_and_normalized_allele_info()` to normalize Indel alleles identically to the reference allele, leading to their removal via `unique_alts.erase(shared_ref)`.

**Impact**:
- Multi-allelic Indel sites (DEL + INS) degraded into pure SNPs, losing the Indel component entirely
- Pure Indel sites (deletions only) were completely absent from the VCF output
- Affected ~2.2% of variants genome-wide in large-scale NIPT data (93,000 samples)

**Fix**: Changed `.bbf` format (v1 → v2) to store `ref_base` per-read instead of per-position. Each read now carries its own `ref_base` with the correct length, preserving the semantic distinction between SNP and Indel reads at the same genomic position.

- `write_binary_record()`: Removed position-level `ref_base`; added per-read `ref_b_len` + `ref_b` bytes
- `read_binary_record()`: Removed position-level `ref_base` read; reads per-read `ref_b_len` + `ref_b` for each read
- `BBF_VERSION` bumped from 1 to 2 (backward-incompatible; old `.bbf` files will be rejected with a clear error message)

**Verification**: Added dedicated unit test (Test 14) that reproduces the exact Indel relocation scenario with mixed SNP/DEL/INS reads. All 105 unit tests pass.

## Upgrade Recommendation

All v2.6.0 users are **strongly recommended** to upgrade to v2.6.1. Existing `.bbf` batchfiles generated by v2.6.0 should be regenerated (delete old `.bbf`/`.bbi` files and re-run with `--smart-rerun` or manually).
